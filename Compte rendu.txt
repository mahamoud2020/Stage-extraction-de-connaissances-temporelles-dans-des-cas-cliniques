
	Test de l'outil CorPipe sur le corpus clinique E3C


Étape 1 : Préparation des données et conversion de format

Problème initial : 

Le corpus E3C est fourni au format XML, tandis que l'outil CorPipe exige un format d'entrée strictement conforme au standard CorefUD (CoNLL-U à 10 colonnes). 
Le parseur natif de l'outil rejetait les données à cause de contraintes syntaxiques fortes.


Travail réalisé : 

J'ai créée un script Python utilisant la bibliothèque cassis pour lire les fichiers XMI, extraire la segmentation (phrases et tokens), et générer des fichiers CoNLL-U d'inférence. 

L'astuce a consisté à générer des colonnes vides (_) pour la syntaxe afin de court-circuiter les contraintes du parseur de CorPipe. J'ai d'abord testé ce script sur un seul fichier XMl pour voir si cela peut fonctionner, je l'ai ensuite appliqué en lot (batch) sur les 81 fichiers XML du corpus.


Étape 2 : Inférence du modèle

Travail réalisé : 

Pour éviter de recharger le modèle (mT5-large affiné par l'UFAL) à chaque fichier, j'ai mis en place un script d'automatisation (subprocess). Cela a permis de charger le modèle une seule fois en mémoire et de traiter l'intégralité du corpus en une passe.

Résultat : 

Le modèle a généré 81 fichiers .conllu de sortie, intégrant ses prédictions de coréférence dans la 10ème colonne (sous le format Entity=(c1...c1)).


Étape 3 : Parsing et extraction des prédictions

Travail réalisé : 

Pour rendre les prédictions du modèle exploitables et faciliter l'évaluation, j'ai écrit un parseur (basé sur des expressions régulières) capable de parcourir la 10ème colonne des fichiers générés. Il gère l'ouverture et la fermeture des balises pour reconstituer les mentions multi-tokens.

Résultat : 

Export de l'ensemble des mentions détectées par le modèle dans un format tabulaire (CSV).


Étape 4 : Première évaluation (Détection de mentions

Travail réalisé : 

J'ai scripté une première évaluation stricte sur la tâche de détection de mentions, en croisant les prédictions du modèle avec les annotations manuelles (la vérité terrain issu des balises CLINENTITY du XMI).


Observation :

Les scores de Précision et de Rappel sont pour l'instant très bas, ce qui peut s'expliquer peut être par une divergence majeure dans les guides d'annotations :

-> Le corpus E3C est annoté de manière très spécifique (uniquement des entités cliniques, maladies, anatomie, etc.)

-> Le modèle mT5-large fait de la résolution de coréférence générale , il peut donc extraire massivement des pronoms ("il", "qui", etc) et des entités non médicales (d'où le taux élevé de faux positifs dans le script ).
